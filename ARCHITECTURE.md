# vibeco 專案架構（最高憲法）

## 總則

- **專案名稱**：vibeco  
- **溝通原則**：前端與後端**嚴格分離**，只透過 **HTTP API** 交換資料；禁止在單一程式庫內混寫「畫面」與「商業邏輯後端」而繞過 API。

## 前端（TypeScript）

- **框架**：Next.js  
- **資料驗證／型別契約**：Zod（用於表單、請求／回應的執行期驗證與型別推導）  
- **職責**：使用者介面、路由、呼叫後端 API、在客戶端／伺服器端依 Zod 驗證資料。

## 後端（Python）

- **框架**：FastAPI  
- **資料模型與驗證**：Pydantic（請求／回應結構、API 契約）  
- **職責**：業務邏輯、資料存取、對外提供 REST／JSON API；不負責前端畫面。

## 邊界

| 層級   | 技術              | 禁止事項                         |
|--------|-------------------|----------------------------------|
| 前端   | Next.js + Zod     | 不寫 Python 後端邏輯於前端專案   |
| 後端   | FastAPI + Pydantic | 不寫 React／HTML 畫面於後端專案 |

## API 契約

- 所有公開端點應有**可文件化**的請求／回應型別（前端用 Zod，後端用 Pydantic），兩邊語意對齊。  
- 錯誤回應（例如驗證失敗 **422**）應回傳**清楚、可對應欄位**的 JSON，供前端顯示錯誤（例如表單欄位紅線）。

## 本地開發埠號（約定）

- 前端：http://localhost:3000  
- 後端：http://localhost:8000  

以上為團隊與 `docker-compose` 預設設定之依據。
